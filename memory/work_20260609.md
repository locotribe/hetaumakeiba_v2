## 未解決バグ：券種別集計の問題

### バグ1（購入金額が過少）— ボックス・フォーメーション
`calculateTypeStats` が `p['購入金額']` をそのまま加算しているが、ボックス/フォーメーションでは `購入金額` = **1点あたりの金額**（合計ではない）。

- 実際の合計 = `購入金額 × 組合せ数`
- `合計金額` フィールド（`parsedTicket['合計金額']`）は正しい合計が入っている
- 修正するなら: `(p['購入金額'] as int? ?? 0) * (p['組合せ数'] as int? ?? 1)`

**How to apply:** 券種別集計の「購入金額」列が実際より少ない場合はこのバグ。ユーザーから金額ズレの指摘があれば修正する。

### バグ2（一部券種が集計されない）— 複数券種混在馬券（調査中・未確定）
ワイドと三連複が混在する1枚の馬券で、一部の券種が券種別テーブルに表示されない現象。

**静的解析の結論:**
- 通常(typeCode="0")混在馬券では `purchases` に複数エントリが作られ、`calculateTypeStats` はすべてカウントするはず（コード上バグなし）
- ながし(typeCode="2")等の単一券種はエントリが1つのみで `countMap += 1` = 1枚扱い（組合せ数に関係なく）

**原因未特定**。次のステップ: 実際の混在馬券の `parsedDataJson` を確認。以下デバッグコードで確認可能:
```dart
debugPrint('purchases length: ${parsedTicket['購入内容']?.length}');
debugPrint('typeIds: ${(parsedTicket['購入内容'] as List).map((p) => p['式別']).toList()}');
```
`purchases.length == 1` なら2枚の別券として保存されている可能性が高い。

**How to apply:** ユーザーから「混在馬券の集計バグ修正」依頼が来た場合、まずデバッグでDB内の実際の構造を確認してから対応する。バグ1（購入金額計算）と合わせて修正することを検討。
