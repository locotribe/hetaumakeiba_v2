# 作業履歴 2026-06-12

## 対応した不具合
1. 阪神芝2200m等でコース平面図が`SizedBox.shrink`になる
   - 原因: `_mapToTrackTypeKey()`が`shiba`を返すが、阪神/京都/新潟/中山は
     `{venue}_shiba_inner_base_coords.json`/`_outer_`しか存在しないためファイルが見つからない。
   - 修正: `lib/services/course_diagram_service.dart`に`_resolveTrackTypeKey()`を追加。
     `CourseElevations.findRaceCourse(venueCode, distance, 'shiba')`が見つからない場合、
     `shiba_inner`→`shiba_outer`の順でフォールバック検索し、実際の`baseData.trackType`で
     coords JSONを読み込む。`getCourseDiagram()`のシグネチャに`raceDistance`引数を追加し、
     `lib/widgets/shutuba_tabs/race_info_tab.dart`の呼び出し箇所も更新。

2. 東京芝1400mで黄色軌跡・S/Gマーカーが画像左上の余白（空中）に表示される
   - 根本原因: `assets/data/course_diagrams/*_base_coords.json`の`edge_points`が
     「ゴールからの距離順」の走路パスではなく、画像のスキャンライン順（y→x順）の
     輪郭点群（芝/ダート/障害境界やラベル枠も混在）になっている。
     東京芝の場合、`base_lap_distance`=2083.1mに対し`edge_points`を順に辿った
     累積距離は約424,000m（約200倍）に達し、`coords.indexAtDistance(1400)`は
     配列のごく早い段階(index 95/2565、座標(132,32))を返す＝画像左上の余白。
     `approach_paths`は現状全JSONファイルで未定義（空）。
   - 暫定対応: `lib/widgets/shutuba_tabs/course_diagram_painter.dart`の`paint()`で、
     `coords.approachPathFor(raceDistance)`が`null`/空の場合は軌跡(黄色Path)と
     S/Gマーカーの描画を`return`でスキップ。outline表示(#1)・高低差グラフは変更なし。
     現状は全JSONで`approach_paths`が空のため、全コースで軌跡/マーカーは非表示になる。
