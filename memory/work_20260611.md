## 出馬表詳細ページ：高低差グラフ消滅の原因調査（v13.41.0リリース後の追加作業）

[[改善案リスト]]に🔴高優先度の進行中タスクとして追加。次回はここから着手する。

### 問題
- DBが再インストールで初期化された状態で、`shutuba_table_cache`が空のまま
  レース結果（`race_result`）のみ存在するレースを開くと、出馬表詳細タブ
  （`lib/widgets/shutuba_tabs/race_info_tab.dart`）の高低差グラフ（fl_chart製）が表示されない。

### 原因
- `race_info_tab.dart`の`_buildElevationChartSection()`(L277〜289)は、
  `predictionRaceData.distanceValue`が`null`だと`SizedBox.shrink()`を返して描画自体をスキップする。
- `shutubaCache == null`かつ`dbResult != null`の場合、`race_page.dart`(L56〜128)と
  `shutuba_table_page.dart`(L306〜346)の`_createPredictionDataFromRaceResult()`が
  `PredictionRaceData`を生成するが、`trackType`/`distanceValue`/`direction`/`courseInOut`を
  一切セットしておらず全て`null`になるのが直接原因。
- 高低差データ本体（`lib/db/course_elevations.dart`＋`lib/db/elevations/<venue>/`配下）は
  Dart定数としてアプリにバンドルされておりDBに依存しないため無傷。検索キーとなる
  距離・トラック種別等の欠落のみが問題。

### 次回着手タスク：案A（共通パーサー実装）
- `raceResult.raceInfo`（例: "ダ右1900m / 天候 : 雨 / ダート : 稍重"）から、
  正規表現 `(芝|ダ|障)(右|左|直)?(\d+)m(?:\((外|内)\))?` で
  `trackType`/`direction`/`distanceValue`/`courseInOut`を抽出する共通関数を実装する
  （`lib/logic/parse.dart`または新規`lib/logic/race_info_parser.dart`）。
- `race_page.dart`と`shutuba_table_page.dart`の両`_createPredictionDataFromRaceResult()`内の
  `PredictionRaceData(...)`生成箇所に、抽出した4項目を追加で渡す。
- 案B（コース図＋高低差グラフの統合表示）は案A完了後に着手予定。
