# 2026-06-10 作業サマリー

## 完了した修正

### 1. parse.dartのウラフィールド欠落バグ修正（v13.39.2）

**バグの原因（手動トレースで特定）:**
JRA QRフォーマットでは typeCode="0"（通常）の全エントリに12バイトが割り当てられる。
- bettingCode(1) + 馬番スロット(6) + 購入金額(5) = 12バイト
- 2頭式別では 馬番(4) + 予備フィールド(2) + 購入金額(5) の構成
- parse.dart は「予備フィールド2バイト」を 単勝(1)・複勝(2)・馬単(6) でのみ読んでいたが、枠連(3)・馬連(5)・ワイド(7) では読んでいなかった

**結果:** ワイド混在馬券で2バイトずれ → 購入金額が"00000"と誤読され0円 → itr位置52='0'が偽の終端と判定されループ早期break → 三連複3エントリが未処理

**修正箇所:** `lib/logic/parse.dart` line 95–96  
```dart
// 修正前
if (bettingCode == "1" || bettingCode == "2" || bettingCode == "6")
// 修正後
if (bettingCode == "1" || bettingCode == "2" || bettingCode == "3" ||
    bettingCode == "5" || bettingCode == "6" || bettingCode == "7")
```
バックアップ: `parse.dart.bak`

**Why:** 枠連・馬連・ワイドにも2バイトの予備フィールドがある。馬単のみ"ウラ"として有意味に使用。

### 2. ticket_aggregator.dartのボックス/フォーメーション購入金額修正（v13.39.3）

**修正箇所:** `lib/logic/ticket_aggregator.dart` `calculateTypeStats()` line 85–86  
```dart
// 修正前
final amount = (p['購入金額'] as int?) ?? 0;
// 修正後
final combCount = (p['組合せ数'] as int?) ?? 1;
final amount = ((p['購入金額'] as int?) ?? 0) * combCount;
```

バックアップ: `ticket_aggregator.dart.bak`

**Why:** ボックス/フォーメーションでは `購入金額` は1点あたりの金額。合計は `購入金額 × 組合せ数`。
